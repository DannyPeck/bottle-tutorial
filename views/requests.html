Requests and request types

We use requests to initiate an action.

For example, retrieve a new view, performing login, retrieve data, update data, delete data, etc.

There are 4 main different types of requests

The type of request along with a route destination should indicate what type of action you wish to perform.

GET

We use a GET request to retrieve data.

For example, if I wanted to view all of the users in my application.

Another example, if I wanted to view an individual user in my application.

Route Example: GET /users or GET /users/:userId

POST

We use a POST request to create data or to send new data.

For example, if I were trying to create a user in an application, I would send the users data in a POST request to be created.

Another example, I want to login to an application, I would send my credentials in a POST request to be verified.

Route Example: POST /users

PUT

I would use a PUT request if I wished to update some existing data.

For example, if I wanted to update my password on my account.

Route Example: PUT /users/:userId

DELETE

I would use a DELETE request if I wished to delete some existing data.

For example, if I wanted to delete my account.

Route Example: DELETE /users/:userId
